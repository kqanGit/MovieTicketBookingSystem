Detailed Description

View model combining seat information with its booking status.

The SeatView class implements the View Model pattern, providing a composite object that combines seat domain data with presentation-specific status information. This class is primarily used by the UI layer to display seat information along with their current booking status.

Key Features:

  • Combines domain object (ISeat) with status information
  • Lightweight wrapper for UI presentation
  • Immutable design after construction
  • Thread-safe for read operations

Design Pattern: View Model

  • Separates presentation logic from domain logic
  • Provides UI-specific data structure
  • Enables clean data binding in user interfaces

Usage Example:

// Create a seat view for UI display
std::shared_ptr<ISeat> seat = seatFactory.createSingleSeat("A1", 100.0);
SeatView seatView(seat, SeatStatus::AVAILABLE);
// Use in UI rendering
if (seatView.status == SeatStatus::AVAILABLE) {
renderAvailableSeat(seatView.seat);
} else {
renderBookedSeat(seatView.seat);
}

Creates a new view model instance that combines the seat domain object with its current booking status for UI presentation.

Parameters
seatShared pointer to the seat entity
statusCurrent booking status of the seat
Precondition
seat must not be null
status must be a valid SeatStatus value
Postcondition
SeatView is initialized with provided seat and status
Exceptions
std::invalid_argumentif seat is null
Note
The constructor creates a lightweight wrapper without copying seat data
The seat object is shared, so modifications to the underlying seat will be reflected in this view

Usage:

auto seat = std::make_shared<SingleSeat>("B5", 150.0);
SeatView view(seat, SeatStatus::AVAILABLE);
// View is ready for UI binding
displaySeat(view);
