Detailed Description

Provides mapping functionality between Movie entities and MovieDTO objects.

The MovieMapper class implements the Data Mapper pattern, providing static methods for converting between domain objects (IMovie) and data transfer objects (MovieDTO). This separation allows for clean data transformation between the business layer and persistence layer while maintaining proper abstraction.

Key Features:

  • Bidirectional mapping between entities and DTOs
  • Static utility methods for clean API usage
  • Thread-safe operations (stateless design)
  • Null-safe implementations
Note
This class follows the static utility pattern - all methods are static and the class cannot be instantiated.

Usage Example:

// Convert entity to DTO for database operations
std::shared_ptr<IMovie> movie = getMovieFromSomewhere();
MovieDTO dto = MovieMapper::toDTO(movie);
// Convert DTO back to entity after database retrieval
MovieDTO retrievedDto = database.getMovie(id);
auto entity = MovieMapper::toEntity(retrievedDto);
// Update existing entity with DTO data
MovieMapper::updateFromDTO(existingMovie, updatedDto);

MovieDTO MovieMapper::toDTO ( const std::shared_ptr< IMovie > & movie)
static

Converts an IMovie entity to a MovieDTO object.

Transforms a domain movie object into a data transfer object suitable for database operations or serialization. All movie properties are mapped to their corresponding DTO fields.

Parameters
movieShared pointer to the movie entity to convert
Returns
MovieDTO object containing the mapped data
Precondition
movie must not be null
Postcondition
Returned DTO contains all data from the input movie
Exceptions
std::invalid_argumentif movie pointer is null
Note
This method is thread-safe and does not modify the input entity

Usage:

std::shared_ptr<IMovie> movie = std::make_shared<Movie>("Inception", 148, "Sci-Fi");
MovieDTO dto = MovieMapper::toDTO(movie);
// dto now contains all movie data for persistence operations

Creates an IMovie entity from a MovieDTO object.

Parameters

Returns
Shared pointer to the newly created movie entity
Precondition
dto must contain valid movie data
Postcondition
Returns a valid IMovie implementation with DTO data
Exceptions
std::invalid_argumentif DTO contains invalid data
Note

The returned object is completely independent of the input DTO

Usage:

MovieDTO dto("Avatar", 162, "Action");

static

Updates an existing IMovie entity with data from a MovieDTO.

Modifies the properties of an existing movie object using the values from the provided DTO. This is useful for updating entities with new data from the database without creating new objects.

Parameters
movieShared pointer to the movie entity to update
dtoThe MovieDTO containing the new data
Precondition
movie must not be null
dto must contain valid movie data
Postcondition
movie contains all updated data from dto
Exceptions
std::invalid_argumentif movie is null or dto is invalid
Note
This method modifies the existing movie object in-place
Thread safety depends on the underlying movie implementation

Usage:

std::shared_ptr<IMovie> movie = getExistingMovie();
MovieDTO updatedData("New Title", 120, "Drama");
MovieMapper::updateFromDTO(movie, updatedData);
// movie now has the updated properties
