Detailed Description

Concrete implementation of movie ticket booking operations.

This class handles all movie ticket booking-related business logic including ticket reservation, booking history management, and seat availability checking. It implements the IBookingService interface and coordinates with booking repositories for data persistence.

Key Features:

  • Movie ticket reservation and booking
  • User booking history retrieval
  • Real-time seat availability checking
  • Multi-seat booking support
  • Integration with showtime management
  • ACID transaction support for booking operations
Design Patterns Used
  • Strategy Pattern: Implements IBookingService interface
  • Repository Pattern: Uses IBookingRepository for data access
  • Dependency Injection: Repository injected via constructor
  • RAII: Uses smart pointers for automatic memory management
Usage Example
auto bookingRepo = std::make_shared<BookingRepositorySQL>();
BookingService service(bookingRepo);
// Create a booking
std::vector<std::string> seats = {"A1", "A2"};
service.createBooking(userId, showtimeId, seats);
// View booking history
auto history = service.viewBookingHistory(userId);

Create a new movie ticket booking.

Creates a booking for specified seats at a given showtime for a user. Handles seat availability validation, booking persistence, and seat status updates atomically.

Parameters
userIDUnique identifier of the user making the booking
showTimeIDUnique identifier of the movie showtime
seatsVector of seat identifiers to reserve (e.g., "A1", "B5")
Precondition
userID > 0 (valid user identifier)
showTimeID > 0 (valid showtime identifier)
!seats.empty() (at least one seat to book)
All seats in the vector are available for booking
Postcondition
Booking record created in database
All specified seats marked as occupied
User's booking history updated
Exceptions
std::invalid_argumentif any seat is already booked
std::runtime_errorif database operation fails
Transaction Behavior
This operation is atomic - either all seats are booked successfully or none are booked if any error occurs.
Warning
Concurrent booking attempts for same seats may conflict
