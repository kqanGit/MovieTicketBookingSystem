Detailed Description

Interface defining booking operations for movie tickets.

This interface provides a contract for all booking-related operations, implementing the Service Layer pattern to encapsulate business logic for ticket reservations and seat management.

The service handles:

  • Creating new bookings with seat selection
  • Retrieving booking history for users
  • Checking seat availability for showtimes
  • Managing booking state transitions

Creates a new booking for specified seats and showtime.

Reserves selected seats for a user at a specific showtime, performing validation and updating seat availability.

Parameters
userIDThe ID of the user making the booking
showTimeIDThe ID of the movie showtime
seatsVector of seat identifiers to be booked
Precondition
userID must be a valid registered user
showTimeID must be a valid future showtime
All seats must be available for booking
Postcondition
Seats are reserved and marked as unavailable
Booking record is created in the system
Exceptions
std::invalid_argumentif userID or showTimeID is invalid
std::runtime_errorif seats are already booked
std::runtime_errorif booking creation fails
Note
This operation should be atomic to prevent double-booking
Warning
This method may block during database transactions
