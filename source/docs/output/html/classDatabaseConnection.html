Detailed Description

Singleton class for managing SQLite database connections.

This class implements the Singleton design pattern to ensure only one database connection exists throughout the application lifecycle. It provides thread-safe access to the SQLite database and manages connection pooling and query execution.

Key Features:

  • Thread-safe singleton implementation
  • Automatic connection management
  • SQLite optimization settings
  • Parameterized query support
  • Connection health monitoring
  • SQL file execution support
Usage Example
DatabaseConnection* db = DatabaseConnection::getInstance();
db->connect("database.db");
// Execute a query
std::vector<std::string> params = {"user123"};
auto results = db->executeQuery("SELECT * FROM users WHERE username = ?", params);
// Execute non-query
bool success = db->executeNonQuery("INSERT INTO users (username) VALUES (?)", params);
